# Описание модулей

# HTML модуль

index.html - основной файл приложения: содержит структуру игрового интерфейса, определяет игровое поле (сетку клеток) и панель управления, подключает CSS стили и JavaScript модули.

# CSS модуль

styles/main.css - стилизация приложения: отвечает за визуальное оформление игрового поля, клеток и инструментов, реализует адаптивную верстку и цветовую схему, определяет анимации и переходы для улучшения UX.

# JavaScript модули

scripts/main.js - точка входа приложения: инициализирует игру, создает экземпляр класса Game.

scripts/Game.js - основной класс управления игровой логикой: управляет созданием и обновлением игрового поля, обрабатывает пользовательский ввод (клики по клеткам и инструментам), координирует взаимодействие между клетками и растениями, запускает и управляет игровым циклом.

scripts/classes/Cell.js - базовый класс клетки игрового поля: определяет свойства клетки (тип, координаты, влажность, растение), реализует расчет влажности на основе соседних водных клеток, управляет посадкой и удалением растений, обеспечивает визуальное отображение клетки.

scripts/classes/Plant.js - абстрактный базовый класс растения: определяет общие свойства всех растений (влажность, стадии роста), реализует базовую логику роста и проверки условий выживания, предоставляет интерфейс для получения информации о растении.

scripts/classes/SwampPlant.js - класс болотника (наследник Plant): определяет специфичные параметры болотника (высокая влажность (0.7-1.0)), реализует уникальную последовательность эмодзи для стадий роста, переопределяет методы получения информации для болотника.

scripts/classes/Potato.js - класс картошки (наследник Plant): определяет параметры картошки (средняя влажность (0.4-0.8)), реализует визуальное представление картошки через эмодзи, демонстрирует принцип наследования от базового класса Plant.

scripts/classes/Cactus.js - класс кактуса (наследник Plant): определяет параметры кактуса (низкая влажность (0.0-0.3), реализует уникальную визуализацию для кактуса, показывает полиморфизм через переопределение методов.
